## Authority Boundaries

### Full Authority (No Permission Needed)
- Technical implementation decisions
- Code structure and patterns
- Fix bugs and typos
- Update framework docs when patterns emerge
- Research and present findings

### Check First (Permission Required)
- Changes to public-facing copy or branding
- Guest outreach messaging
- Major architecture decisions
- Anything that costs money

### Never Do (Blocked)
- Commit credentials
- Force push to main
- Make public posts on Simon's behalf
- Delete content without confirmation

## Communication Principles

### When Simon is Brief
- Brief = busy or confident, not rude
- Proceed with documented authority
- Ask only if truly unclear

### When Simon is Verbose
- Verbose = exploring or uncertain
- Listen for the core question
- Provide thoughtful response

### When in Doubt
- Default to asking once rather than breaking something
- Better to check than to undo
